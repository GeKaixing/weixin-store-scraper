# Session: 2026-05-11 微信小店达人广场爬取迭代

## 概述
本次会话完成了微信小店达人广场爬虫从一趟式（v11 one-pass）到两趟方案的重构，修正了详情页字段解析、翻页失效和分页页码提取问题。

## 关键发现

### 1. 开标签取详情 → Vue 状态被污染 → 翻页失效
- **现象**：点 18 个"详情"开标签 → 关标签后，输入框+跳转按钮翻页到第 2 页，但数据和第 1 页完全一样
- **原因**：Vue SPA 组件状态被 18 次 `window.open` 原生行为（或 `ctx.expect_page`）破坏
- **根因确认**：即使不用 `window.open` 覆盖，`ctx.expect_page` 开真实标签页也会污染状态
- **解决方案**：两趟分离——第一趟用 `window.open` 覆盖（返回假窗口）取 URL，不污染状态；第二趟用 `page.goto` 独立爬详情

### 2. 详情页文本解析：标签和价值分行
- 旧的 `DETAIL_PARSE_JS` 假设标签和价值在同一行（如 `"带货销售额 ￥50万-100万"`）
- 实际 DOM 中它们在不同行：
  ```text
  带货销售额      ← line i
  ￥50万-100万    ← line i+1
  ```
- **带货概览** 修正：`l === '带货销售额'` 时读 `lines[i+1]`
- **粉丝特征** 修正：四行一组（标签/值/占/百分比），`粉丝性别 = lines[i+1] + ' ' + lines[i+3]`
- **品类占比** 修正：用 `lines[i+1].endsWith('%')` 替代被 patch 工具搞乱的正则

### 3. `get_max_page` 误解析 1234
- 分页文本 `"1234...250下一页跳转"` 
- `match(/\d+/g)` → `["1234", "250"]` → `Math.max(1234, 250) = 1234`
- 正确方式：`text.match(/\.\.\.(\d+)/)` 提取省略号后数字 = 250

### 4. `patch` 工具转义符翻倍问题
- 多次使用 `patch` 修改 `DETAIL_PARSE_JS` 时，其中的 `\n` 和 `\d` 等转义符被逐层翻倍
- `\\n` → `\\\\n` → `\\\\\\\\n`
- `\d` → `\\d` → `\\\\d`
- 现象：正则 `endsWith('%')` 的修复使用纯字符串方法，避免了转义符问题
- 教训：对于复杂 JS 模板字符串中的正则，用字符串方法替代或用 `write_file` 全量覆盖

### 5. 全量爬取 6 小时 + 网络断连
- 保健食品/膳食营养补充食品：168 页，3254 条
- 第一趟：~8 分钟
- 第二趟：爬了 ~2220 条后网络断开，剩下 ~1034 条无详情数据
- 临时 JSON 每 10 条保存一次，网络断开后丢失了部分已爬数据（JSON 文件保存的最后 10 条 checkpoint）
- 建议：第二趟每 5 条保存一次，或增加断点续爬功能
